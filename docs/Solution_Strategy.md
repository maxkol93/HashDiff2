# Solution Strategy: HashDiff

## Problem Analysis (First Principles)

**Корневая проблема:** Сравнение двух файлов требует нескольких шагов (открыть консоль, ввести команду, скопировать хеши, сравнить глазами). Это прерывает рабочий процесс.

**Разложение на атомы:**
1. Нужно получить MD5 двух файлов → `hashlib.md5()`
2. Нужно сравнить два значения → строковое сравнение
3. Нужно показать результат наглядно → цвет строки
4. Нужно убрать friction загрузки файлов → drag & drop

**Решение:** Минимальное приложение, которое делает только это — не больше.

## Architecture Decision

### Single-file Application
Одна точка входа `app.py`, никаких серверов, никаких баз данных. Всё в памяти.

### State Model
```
AppState:
  left_files: List[FileEntry]
  right_files: List[FileEntry]

FileEntry:
  path: str
  name: str
  md5: str | None  (None = вычисляется)
  status: 'pending' | 'hashing' | 'done' | 'error'
```

### Matching Algorithm
1. Попытаться сопоставить по имени файла (без пути)
2. Если нет пары по имени — по порядку добавления
3. Файл без пары — статус 'unpaired' (серый)

### Hashing Strategy
- Вычислять MD5 в отдельном потоке (`threading.Thread`)
- UI не блокируется
- Прогресс-бар для файлов > 10 MB
- Streaming: читать по 8192 байт

## TRIZ Application
**Противоречие:** Большие файлы требуют времени → пользователь ждёт.
**Решение (разделение во времени):** Показывать UI немедленно, хешировать в фоне, обновлять результат по готовности.

## Risk Mitigation
| Risk | Mitigation |
|------|------------|
| Антивирус блокирует exe | Инструкция по добавлению в исключения |
| tkinterdnd2 не работает | Fallback на кнопку «Открыть» |
| Большой файл зависает UI | Threading + asyncupdate |
| Кириллика в путях | Использовать pathlib.Path везде |
