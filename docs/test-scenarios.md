# Test Scenarios: HashDiff (BDD)

## Feature: File Comparison

### Happy Path

```gherkin
Scenario: Сравнение двух идентичных файлов
  Given приложение HashDiff запущено
  When пользователь перетаскивает "report.pdf" в левую панель
  And пользователь перетаскивает копию "report.pdf" в правую панель
  And MD5 вычисление завершено
  Then строка пары подсвечена зелёным (#d4edda)
  And отображается текст "✓ Идентичны"
  And MD5 хеши обоих файлов одинаковы

Scenario: Сравнение двух разных файлов
  Given приложение HashDiff запущено
  When пользователь перетаскивает "file_v1.zip" в левую панель
  And пользователь перетаскивает "file_v2.zip" в правую панель
  And MD5 вычисление завершено
  Then строка пары подсвечена красным (#f8d7da)
  And отображается текст "✗ Отличаются"
  And MD5 хеши отличаются
```

### Error Handling

```gherkin
Scenario: Файл недоступен для чтения
  Given приложение HashDiff запущено
  When пользователь добавляет файл без прав на чтение
  Then статус файла показывает "⚠ Нет доступа"
  And строка не участвует в сравнении (серая)

Scenario: Файл удалён во время хеширования
  Given файл начал хешироваться
  When файл удаляется с диска в процессе
  Then статус показывает "⚠ Файл не найден"
  And приложение не крашится

Scenario: Drop папки вместо файла
  When пользователь перетаскивает папку в панель
  Then показывается уведомление "Папки не поддерживаются"
  And папка не добавляется в список
```

### Edge Cases

```gherkin
Scenario: Файл размером 0 байт
  Given пользователь добавляет пустой файл в обе панели
  When MD5 вычисление завершено
  Then MD5 хеш равен "d41d8cd98f00b204e9800998ecf8427e"
  And файлы считаются идентичными (зелёный)

Scenario: Умное выравнивание по имени файла
  Given в левой панели файлы: [alpha.txt, beta.zip, gamma.pdf]
  And в правой панели файлы: [gamma.pdf, alpha.txt, beta.zip]
  When пары вычисляются
  Then alpha.txt выровнен с alpha.txt
  And beta.zip выровнен с beta.zip
  And gamma.pdf выровнен с gamma.pdf

Scenario: Файл без пары
  Given в левой панели 3 файла
  And в правой панели 2 файла
  When пары вычисляются
  Then третий файл слева показан серым
  And статус "— Нет пары"

Scenario: Кириллические имена файлов
  Given файл с именем "отчёт_2024.pdf" в левой панели
  And копия файла в правой панели
  When MD5 вычисление завершено
  Then файлы корректно сопоставлены по имени
  And результат зелёный
```

### Security

```gherkin
Scenario: Файл за пределами разрешённых директорий
  Given пользователь пытается добавить системный файл (C:\Windows\System32\...)
  When файл добавлен в панель
  Then хеш вычисляется нормально (read-only операция, безопасно)
  And файловая система не модифицируется

Scenario: Очень большой файл (>4GB)
  Given файл размером 5 GB добавлен в панель
  When вычисление идёт
  Then UI остаётся отзывчивым
  And прогресс-индикатор обновляется
  And память не исчерпывается (streaming)
```
