# Specification: HashDiff

## User Stories & Acceptance Criteria

### US-01: Drag & Drop файла
**As a** технический пользователь  
**I want to** перетащить файл в левую или правую панель  
**So that** мне не нужно вводить путь вручную  

**Acceptance Criteria (Gherkin):**
```gherkin
Feature: Drag and Drop

  Scenario: Успешный drop файла в левую панель
    Given приложение открыто
    When пользователь перетаскивает файл в левую панель
    Then файл появляется в списке левой панели
    And начинается вычисление MD5

  Scenario: Drop нескольких файлов
    Given приложение открыто
    When пользователь перетаскивает 3 файла в левую панель
    Then все 3 файла появляются в списке
    And для каждого начинается вычисление MD5

  Scenario: Drop папки (не поддерживается в MVP)
    When пользователь перетаскивает папку
    Then показывается уведомление "Папки не поддерживаются. Перетащите файлы."
```

### US-02: MD5 вычисление
**As a** пользователь  
**I want to** видеть MD5 хеш каждого файла  
**So that** я могу самостоятельно проверить значения  

**Acceptance Criteria:**
```gherkin
  Scenario: Вычисление хеша обычного файла
    Given файл загружен в панель
    When вычисление завершено
    Then отображается MD5 строка (32 символа, hex)
    And время вычисления < 2 секунд для файлов < 100 MB

  Scenario: Вычисление хеша большого файла
    Given файл размером > 100 MB
    When вычисление идёт
    Then UI не зависает
    And показывается индикатор прогресса

  Scenario: Файл заблокирован системой
    When пользователь добавляет файл который нельзя прочитать
    Then показывается статус "Ошибка: нет доступа"
    And MD5 не вычисляется
```

### US-03: Визуальное сравнение
**As a** пользователь  
**I want to** сразу видеть совпадают ли файлы  
**So that** мне не нужно сравнивать строки хешей вручную  

**Acceptance Criteria:**
```gherkin
  Scenario: Одинаковые файлы
    Given в левой панели файл A и в правой файл B
    When MD5(A) == MD5(B)
    Then строка пары подсвечена зелёным
    And показан текст "✓ Идентичны"

  Scenario: Разные файлы
    Given в левой панели файл A и в правой файл B
    When MD5(A) != MD5(B)
    Then строка пары подсвечена красным
    And показан текст "✗ Отличаются"

  Scenario: Файл без пары
    Given в левой панели 3 файла, в правой 2 файла
    Then третий файл слева показан серым
    And показан текст "— Нет пары"
```

### US-04: Сопоставление по имени
**As a** пользователь  
**I want to** чтобы файлы с одинаковым именем автоматически выровнялись в одну строку  
**So that** сравнение было наглядным  

**Acceptance Criteria:**
```gherkin
  Scenario: Совпадение по имени файла
    Given в левой панели [report.zip, data.csv]
    And в правой панели [data.csv, report.zip]
    Then report.zip слева выровнен с report.zip справа
    And data.csv слева выровнен с data.csv справа

  Scenario: Нет совпадения по имени
    Given в левой панели [fileA.txt]
    And в правой панели [fileB.txt]
    Then пара создаётся по порядку (fileA ↔ fileB)
```

### US-05: Открытие через диалог
**As a** пользователь  
**I want to** открыть файл через стандартный диалог  
**So that** у меня есть альтернатива drag & drop  

**Acceptance Criteria:**
```gherkin
  Scenario: Кнопка "Добавить файл"
    When пользователь нажимает "Добавить файл" в левой панели
    Then открывается стандартный диалог выбора файла Windows
    And после выбора файл добавляется в панель
```

### US-06: Очистка панели
**As a** пользователь  
**I want to** очистить панель и начать заново  
**So that** я могу сравнивать разные наборы файлов  

**Acceptance Criteria:**
```gherkin
  Scenario: Кнопка "Очистить"
    Given в панели есть файлы
    When пользователь нажимает "Очистить"
    Then все файлы удаляются из панели
    And результаты сравнения сбрасываются
```

## Non-Functional Requirements

| NFR | Requirement |
|-----|-------------|
| Performance | Хеш файла 1 GB < 30 сек |
| Reliability | Не крашиться при недоступном файле |
| Usability | Запуск без установки, одиночный .exe |
| Compatibility | Windows 10/11, x64 |
| Size | exe < 30 MB |
| Encoding | Поддержка Unicode/кириллики в путях |
